# 演示数据设计

## 设计目标

演示数据用于让默认 H2 模式和 Docker MySQL 模式启动后都能直接展示完整的运营状态。初始化内容覆盖车站、车次、库存、订单、支付流水、退款流水、风险事件、风险处置历史、操作日志和 Outbox 事件，避免管理台出现大面积空白。

初始化逻辑位于 `DemoDataInitializer`，按模块分段执行，使用唯一业务编号和固定 `DEMO` 前缀保证幂等。重复启动不会重复写入同一批订单、支付流水、退款流水或 Outbox 事件。

## 初始化开关

默认配置：

```yaml
railway:
  demo-data:
    enabled: true
```

如需使用空库启动，可将 `railway.demo-data.enabled` 设置为 `false`。用户账号初始化不受该开关影响，系统仍会保留 `admin`、`risk`、`ops` 三个默认账号。

## 车站和车次数据

默认初始化 17 个车站，覆盖北京、上海、广州、深圳、杭州、南京、武汉、西安、成都、重庆、郑州、长沙、合肥、天津、济南、南昌和福州。

默认初始化 16 趟车次，覆盖京沪、沪京、京杭、广武、深广、成渝、西郑、武成、沪福、京长、长广、津沪、济南至南昌、合肥至上海等线路。

## 库存数据

系统为全部演示车次生成未来 14 天座位库存：

- 每趟车至少包含二等座库存。
- 高铁车次包含一等座。
- 部分车次包含商务座。
- 库存数量按车次、日期和座席类型做差异化分布。
- 部分库存保留 1 到 5 张的低余票效果，用于展示紧张票源。
- 部分库存保留较高余票，用于展示正常可售车次。

## 订单状态分布

默认生成 48 条订单，覆盖：

- `PENDING_PAYMENT`
- `PAID`
- `CLOSED`
- `REFUNDED`

订单分布在多个用户、多个车次、多个乘车日期和多个创建时间上。`PENDING_PAYMENT` 和 `PAID` 订单会锁定库存，`CLOSED` 和 `REFUNDED` 订单不继续占用库存。

## 支付流水分布

每条演示订单都会配套支付流水，覆盖：

- `PENDING`
- `SUCCESS`
- `FAILED`

`PAID` 和 `REFUNDED` 订单对应 `SUCCESS` 支付流水，并填充内部支付流水号、模拟渠道流水号、回调请求号和支付时间。待支付订单对应 `PENDING` 支付流水，关闭订单对应失败支付记录。

## 退款流水分布

`REFUNDED` 订单会配套退款流水，覆盖：

- `PENDING`
- `SUCCESS`
- `FAILED`

退款流水使用 `RFDEMO` 前缀，成功退款记录包含模拟渠道退款流水号和退款完成时间。退款状态和订单状态分离，体现票务退票与资金退款的独立生命周期。

## 风险事件和处置历史

默认初始化 32 条风险事件，覆盖：

- `PENDING`
- `CONFIRMED`
- `FALSE_POSITIVE`
- `CLOSED`

风险类型覆盖快速购票、高金额订单和频繁退票。非待处理风险事件会生成对应处置历史，记录状态流转、处理人、处理时间和处置备注。

## 操作日志

默认初始化 66 条操作日志，覆盖：

- 登录
- 下单
- 支付
- 关闭订单
- 退票
- 风险处置
- 支付回调
- 退款回调
- Outbox 派发
- 缓存清理

日志时间分布在最近多个日期，便于操作日志页面直接展示审计轨迹。

## Outbox 事件

默认初始化 10 条 Outbox 事件，覆盖：

- `PENDING`
- `PROCESSING`
- `DONE`
- `FAILED`

事件类型覆盖订单支付、订单退票、支付成功、退款创建、退款成功、风险事件处理等场景。少量失败事件用于展示事件中心的失败原因和手动重试按钮。演示 `PENDING` 事件的 `nextRetryAt` 设置为未来时间，避免影响测试中手动构造的 Outbox 派发场景。

## H2 和 Docker MySQL 模式差异

默认 H2 模式和 Docker MySQL 模式使用同一套初始化逻辑。

- H2 模式适合本地快速启动和自动化测试。
- Docker MySQL 模式适合保留运行过程中的演示数据。
- 如果需要重新生成 Docker MySQL 演示数据，可执行 `docker compose down -v` 清理数据卷后重新启动。

## 重置演示数据

H2 内存数据库在应用重启后会重新初始化。

Docker MySQL 模式会使用持久化 volume。清理方式：

```powershell
docker compose down -v
docker compose up --build
```

如果只想关闭演示数据初始化，可在环境变量或配置文件中设置：

```yaml
railway:
  demo-data:
    enabled: false
```
