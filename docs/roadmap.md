# 后续开发路线

## 阶段一：运行与验证

- 安装 Maven。
- 启动 Spring Boot 后端。
- 打开前端页面完成一次查车、下单、支付、退票。
- 检查 H2 控制台中的订单、库存、支付流水和风险事件数据。
- 执行 Maven 集成测试和前端脚本语法检查。

## 阶段二：基础能力增强

- 完善接口异常场景测试。
- 增加更多订单状态非法流转用例。
- 增加支付失败、重复回调和重复下单的边界测试。
- 将 H2 演示数据与 MySQL profile 的初始化策略整理清楚。
- 完善接口文档和数据库字段说明。

## 阶段三：工程化增强

- 完善 Redis 部署示例、缓存监控和限流指标看板。
- 增加 refresh token、登录失败限制和令牌失效机制。
- 增加接口压测和并发购票验证脚本。
- 配置更完整的 GitHub Actions 自动测试流程。
- 增加 Docker Compose 部署说明。

## 阶段四：运营能力增强

- 为风险事件增加风险等级筛选和处理人筛选。
- 增加风险处理 SLA 和超时提醒。
- 增加订单、支付流水和风险事件导出。
- 增加风险趋势图和运营日报。
- 扩展风险规则配置化能力。
