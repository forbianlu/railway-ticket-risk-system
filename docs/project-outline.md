# 铁路客运票务与风控运营管理系统项目大纲

## 1. 项目背景

铁路客运系统涉及车次计划、座位库存、订单交易、退改签、异常行为识别和运营统计。对于铁路局信息技术岗，该项目能体现对交通客运业务的理解；对于银行科技岗，该项目也能体现交易系统、风控规则、日志审计和数据看板能力。

## 2. 项目目标

第一阶段目标是完成一个能运行、能演示、能写进简历的 MVP：

- 乘客可以查询车次和余票
- 乘客可以创建购票订单
- 订单创建后扣减余票
- 乘客可以退票并释放余票
- 系统能识别简单异常交易行为
- 后台可以查看运营统计和风险事件
- 操作过程可记录审计日志

## 3. 用户角色

- 乘客：查询车次、购票、退票、查看自己的订单
- 运营人员：维护车次、查看订单和客流统计
- 风控人员：查看风险事件、分析异常订单
- 管理员：管理用户、角色、权限和日志

## 4. 核心业务模块

### 4.1 基础数据

- 车站管理：车站编码、车站名称、城市、状态
- 车次管理：车次号、始发站、终到站、出发时间、到达时间
- 座位库存：车次、日期、座位类型、总座位、剩余座位

### 4.2 票务交易

- 车次查询：按出发站、到达站、乘车日期查询
- 创建订单：校验余票，扣减库存，生成订单
- 订单查询：按用户、状态、日期查询
- 退票：更新订单状态，释放库存

### 4.3 风控识别

第一阶段使用规则引擎式判断：

- 短时间多次购票：同一用户 10 分钟内下单超过 3 次
- 频繁退票：同一用户 7 天内退票超过 3 次
- 高价值异常：同一用户单日订单金额超过阈值

风控结果包括：

- 风险类型
- 风险等级
- 触发原因
- 关联订单
- 处理状态

风控实现采用规则引擎结构：

- `RiskRule` 定义统一规则接口
- `RiskScene` 区分下单后、退票后等触发场景
- `RiskService` 负责调度规则并生成风险事件
- 每条规则独立实现，便于后续扩展

### 4.4 运营看板

- 总订单数
- 有效订单数
- 退票订单数
- 风险事件数
- 热门车次
- 近 7 日订单趋势

### 4.5 日志审计

- 下单日志
- 退票日志
- 风控触发日志
- 管理员操作日志
