# 前端管理台设计说明

## 1. 设计目标

前端管理台用于集中展示铁路客运票务、支付退款、风险运营、缓存限流和 Outbox 事件运行情况。本轮重构目标是提升信息层级、数据可读性和操作效率，使页面更接近真实后台管理系统。

## 2. 设计原则

- 信息优先：运营看板优先展示订单、风险、退票率、热门车次和系统治理指标。
- 低噪声视觉：使用浅色工作区、深色侧边导航、清晰表格和克制的状态色。
- 高密度但不拥挤：指标卡、筛选区、表格和事件列表按业务分组排列。
- 状态明确：订单、支付、退款、风险和事件状态使用文字和语义色共同表达。
- 可访问性：保留可见 label、键盘焦点、足够对比度和移动端可操作尺寸。
- 响应式：桌面使用侧边导航和多列看板，窄屏下自动折叠为单列布局。

## 3. 页面结构

- 登录屏：展示系统定位、主要能力和演示账号，登录表单集中在右侧。
- 侧边导航：按运营看板、车次查询、订单、支付、退款、风险、事件、日志分组。
- 顶部状态栏：显示当前工作区标题、登录角色和退出入口。
- 内容工作区：每个业务模块独立成段，避免在单个页面堆叠无关操作。

## 4. 核心模块改造

### 登录页

登录页从侧边栏小表单升级为完整入口页，展示系统名称、交易链路、风险运营和系统治理能力，并保留 `admin`、`risk`、`ops` 演示账号说明。

### 运营看板

看板展示总订单、待支付、已支付、已关闭、退票订单、退票率、风险率和未处理风险，并增加热门车次进度条、缓存指标和限流规则展示。

### 订单 / 支付 / 退款

交易相关模块保留原有筛选、分页和操作能力。金额、车次、状态和低库存数据使用更醒目的展示方式，便于快速扫读。

### 风险管理

风险事件列表保留处置按钮和处置历史入口，右侧风险报表展示状态分布和处置比例，便于观察风险运营压力。

### Outbox 事件中心

事件中心突出展示总事件、待处理、处理中、已完成、已失败、失败率和积压数量，并保留单条重试、批量重试和手动派发能力。

### 缓存与限流

缓存模式、TTL、命中、未命中、失效、本地降级、限流模式和拦截次数集中在看板中展示，限流规则以紧凑列表呈现。

## 5. 视觉系统

- 主色：铁路运营感较强的深绿色，用于主按钮、关键状态和品牌识别。
- 辅助色：蓝色用于信息聚合，琥珀色用于待处理状态，红色用于风险和失败状态。
- 背景：浅灰工作区，降低长时间浏览疲劳。
- 组件：指标卡、表格、事件项、筛选区采用统一 8px 圆角和轻量边框。
- 字体：使用系统字体栈，数字列启用 tabular figures，减少数据跳动。

## 6. 交互与响应式

- 导航点击后高亮当前模块。
- 关键按钮具备 hover、focus 和 disabled 状态。
- 窄屏下侧边导航切换为顶部块状导航，表格保留横向滚动容器。
- 支持 `prefers-reduced-motion`，降低动效对敏感用户的影响。

## 7. 未改变的范围

- 未修改后端接口。
- 未修改数据库结构。
- 未引入 React、Vue、Angular 或构建工具。
- 未改变订单、支付、退款、风险、缓存、限流和 Outbox 业务规则。
